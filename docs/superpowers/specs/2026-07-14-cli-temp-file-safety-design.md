# CLI 临时文件安全与日志最小化设计

## 背景

CLI 当前在每次 SSH 连接启动时扫描系统临时目录，并删除不属于当前 PID 的
`ssh-cmd-*.ps1` 和 `ssh-cmd-*.sh`。并发连接因此可能删除其他连接正在执行的脚本。

临时文件还只使用 PID 命名，并通过覆盖写入创建。PID 复用或同一进程内重复创建时，
可能发生文件碰撞。调试日志则会记录完整参数、远程命令和最终命令行，可能泄露 token、
路径或其他敏感参数。

## 目标

- 每个连接只负责删除自己创建的临时脚本。
- 临时脚本名称不可预测且不依赖 PID。
- 文件创建必须具有排他性，不能覆盖已有文件。
- 启动清理只能删除确认已过期的残留脚本。
- 默认日志不得包含远程命令或完整命令行。
- 保持现有路由、SFTP 和子进程启动行为不变。

## 非目标

- 不提供可重新开启完整命令日志的配置项。
- 不实现远程命令内容脱敏规则。
- 不改变配置文件结构或图形界面。
- 不保证进程被强制终止时立即删除临时文件；该场景由启动期过期清理兜底。

## 方案

### 临时文件所有权

在 `crates/cli/src/temp.rs` 中引入 `TempScript`，由它负责创建、持有和删除单个临时脚本。

- 文件名格式为 `ssh-cmd-<GUID><扩展名>`。
- 使用 `OpenOptions::create_new(true)` 原子创建文件，禁止覆盖已有路径。
- 创建后写入远程命令原文，并保存路径供路由模板渲染。
- `TempScript` 实现 `Drop`，离开作用域时仅删除自身持有的文件。
- GUID 极低概率碰撞时重新生成并重试；其他 I/O 错误直接返回。

Windows 主流程拆分为返回退出码的内部函数。顶层 `main` 仅在内部函数返回后调用
`process::exit`，确保临时文件守卫先完成析构。SFTP 和交互式连接不创建临时脚本。

### 启动期过期清理

保留启动期清理作为异常终止后的兜底，但不再接收或比较 PID。

- 只检查系统临时目录顶层的普通文件。
- 只匹配 `ssh-cmd-*.ps1` 和 `ssh-cmd-*.sh`。
- 仅删除修改时间早于当前时间 24 小时以上的文件。
- 无法读取目录、元数据或修改时间时跳过该文件。
- 删除失败时静默忽略，避免清理问题阻断 SSH 连接。

清理函数接收可测试的目录、当前时间和过期阈值参数；生产入口使用系统临时目录、
当前时间和 24 小时阈值。

### 日志最小化

默认日志保留诊断连接和路由所需的最小信息：

- 连接分隔标记。
- SSH 服务端端口。
- 匹配到的路由名称；SFTP 记录固定类型。
- 配置加载、路由匹配、临时文件创建和子进程启动错误。
- 子进程退出码。

删除以下日志：

- 完整 `args`。
- 原始远程 `command`。
- 完整 `cmdLine`。

错误日志只记录错误类型和系统错误，不拼接远程命令内容。日志文件位置和时间格式保持不变。

## 错误处理

- 临时脚本创建或写入失败时记录不含命令内容的错误，关闭 Job Object，并返回失败退出码。
- 临时脚本删除失败不改变子进程退出码。
- 过期清理中的单文件失败不影响其他文件，也不影响当前连接。
- GUID 碰撞只触发有限次数重试；重试耗尽后按创建失败处理。

## 测试

`temp` 模块增加以下测试：

- 新创建的匹配文件不会被过期清理删除。
- 修改时间超过 24 小时的匹配文件会被删除。
- 不匹配命名规则的文件不会被删除。
- `TempScript` 创建的文件名包含 GUID，并保留指定扩展名。
- 已存在路径不会被覆盖。
- `TempScript` 离开作用域后只删除自身文件。

CLI 流程通过提取日志消息生成或可测试的运行逻辑，验证日志不包含原始远程命令、
完整参数或最终命令行。现有路由、配置和 WSL 路径测试继续通过。

## 验收标准

- 两个并发连接不能删除或覆盖彼此正在使用的临时脚本。
- PID 复用不会导致临时文件碰撞。
- 24 小时内的残留脚本不会在启动时被删除。
- 正常完成的连接在退出前删除自身临时脚本。
- 默认日志中不出现远程命令原文或完整执行命令行。
- `cargo test --workspace` 通过。
